# nagi — プロダクト仕様

## ビジョン

**「自然に触れる」体験を iOS 上で再現するマインドフル・アプリ。**

ユーザーが画面の自然（焚き火・波・森など）に**触れると、視覚・触覚・音が同時に反応する**。視覚単体のアンビエント映像アプリや、音声単体の瞑想アプリと差別化する核は**マルチセンサリー・インタラクション**。

## コアコンセプト

> 見るだけの自然映像は飽和している。**触れる自然**はまだ少ない。

1. **Photoreal（写実的）** — 抽象的な宇宙・幾何学ではなく、実在の自然
2. **Reactive（反応する）** — タッチ / ジェスチャー / 呼吸に映像・音・触覚が応答
3. **Multisensory（三位一体）** — Metal（視覚）× Core Haptics（触覚）× PHASE（空間オーディオ）
4. **Long-session friendly** — 10〜30 分の連続使用に耐える（刺激的でない、発熱しない）

## インタラクション契約

- **タッチ → 反応まで**: 視覚反応 ≤ 50ms、触覚発火 ≤ 30ms、音発火 ≤ 50ms（体感同時性の閾値）
- **タッチ位置**: Metal シェーダーに uniform（正規化座標）、PHASE の音源位置、Haptics のトリガ
- **ジェスチャー**: タップ / 長押し / ドラッグ の 3 種。ピンチやスワイプは当面使わない（瞑想時に指を動かしすぎない）

## 実装パイプライン（リアル焚き火を例に）

完全 iOS ネイティブ・procedural（Blender / 動画ベースは採用しない）:

1. **炎本体** — Metal compute シェーダーで FBM ノイズ × 色温度勾配（根元=白黄 2500K → 先端=赤橙 1500K）を描画
2. **火の粉** — GPU パーティクル（Metal compute で position/lifetime 管理、pointsprite で blit）
3. **煙** — 追加の FBM パス（上方フロー + 低周波 + α fade）
4. **薪 / 石 / 地面** — 2D procedural shader（木目・石肌を擬似 3D 風に描く）。USDZ は使わない
5. **熱歪み** — SwiftUI `.distortionEffect` でフルスクリーン distortion（軽量）
6. **Bloom** — Metal 内で luminance 抽出 → Gaussian blur → composite（既存 `fluid_render` 内と同等の ACES tonemap 流用）
7. **音** — `SpatialAudioEngine`（PHASE）で焚き火位置に定位、タップ位置との距離で音量・パン + `AudioEngine`（既存 AVAudioPlayer）で背景環境音ループ
8. **触覚** — `HapticEngine`（既存 + 拡張）で named pattern を事前ロード、タップで発火。PHASE と SyncClock で 30ms 以内の同期

## マイルストーン

1. **PoC**: 焚き火 1 シーンで「触ったら気持ちいい」を検証（視覚 × 触覚 × 音の三位一体がハマるかの判断）
2. PoC がハマれば → Sleep の波、Breath の森 / 夜空 へ展開
3. シーン 3 つが出揃ったらメニュー構成を再検討（現 Tab 構成が最適かも判断材料にする）
